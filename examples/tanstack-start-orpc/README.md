# TanStack Start + oRPC + LiveStore + TanStack DB Todo App

Standalone example that combines:

- **TanStack Start** (file-based routing via `@tanstack/start` + TanStack Router, full-document SSR)
- **oRPC** (`@orpc/server` + `@orpc/client`, contract-first with `posts.*` procedures)
- **LiveStore** (`@livestore/livestore` + `@livestore/adapter-web`, local-first sync)
- **TanStack DB** (via `@cyberistic/livestore-tanstack-db`, useTable + useCrud)
- **Prisma** (single source of truth for the `Todo` model)

## Architecture

```text
Browser client
├── /api/rpc/* ──> TanStack Start oRPC handler
└── LiveStore event log ──> /sync ──> SyncBackendDO (SQLite)
                                         └── event mirror ──> Cloudflare D1
```

The oRPC handler receives typed application mutations. LiveStore records the
same client-side changes locally, synchronizes them through the Durable Object,
and mirrors each pushed event into D1 for inspection and replay.

## Local development

```bash
bun install
bun run db:generate
bun run dev                # http://localhost:60001

bun run typecheck
bun run build
```

## Deploy

Authenticate Alchemy with Cloudflare, then provision D1, the LiveStore sync
Durable Object, and the TanStack Start Worker:

```bash
bun run deploy
```

Remove all provisioned resources with `bun run destroy`.

For ad-hoc event-log mirroring, export a LiveStore `eventlog.db` and run
`bun scripts/d1-mirror.mjs <path-to-eventlog.db> <store-id>`. Add `--local` to
write to Wrangler's local D1 instead of the deployed database.

## Tier demos

| Tier | What it shows | Where |
| ---- | ------------- | ----- |
| 0.6  | TanStack DB commits round-trip through the oRPC client | `useTable('Todo', { rpc: { client, config } })` in Header/MainSection/Footer |
| 1.7  | Bulk insert collapses N rows into one `v1.TodoBulkUpserted` event | `useCrud('Todo').bulkUpsert(...)` in Header + index.tsx |
| 3.1  | `useTable(name)` reads from `<LiveStoreProvider>` (no `liveStore` prop) | `__root.tsx` mounts the provider; child components call `useTable('Todo')` |

## Caveat: `createLiveStoreProvider` doesn't exist yet

The example uses `<LiveStoreProvider schema={...} oRPC={...}>` from
`@cyberistic/livestore-tanstack-db`. The package's `LiveStoreProviderProps`
currently types `schema: LiveStoreSchema` (the `makeSchema(...)` output),
but `useTable` reads `{ store, tables, events, schema }` from the provider's
context — i.e. it expects the wider LiveStore runtime object, not the
`makeSchema` value.

We bridge this by passing `{ store, tables, events, schema }` as the
provider's `schema` prop via `as unknown as ...`. The runtime is built
inside `ClientOnlyLiveStore` in `__root.tsx` once `useAppStore()` resolves.

A future `createLiveStoreProvider(...)` helper would lock this shape in
(see `dream-list.md` Tier 3.1 in the repo root).

## SSR strategy

LiveStore's web adapter requires `navigator.locks` (browser-only). The
example's `__root.tsx` gates the entire `<LiveStoreProvider>` mount
behind a `useEffect`-driven hydration flag, so the server emits the
HTML shell + the loader's oRPC data, and the client takes over to
mount LiveStore and attach the interactive collection hooks.

The route loader (`createFileRoute('/').loader`) calls the server-side
oRPC client (`createRouterClient(router, ...)`) so SSR fetches the
initial list without an HTTP loopback.
